Deployment Guide

Concept Master on a Droplet

Run the app on a Digital Ocean Droplet and access it from any device on your Tailnet.

0 Prerequisites

Before you start, make sure you have:

  1. A running Digital Ocean Droplet (Ubuntu 22.04+ recommended) with SSH access.
  2. A Tailscale account and at least one other device already on your Tailnet.
  3. The Concept Master repo cloned onto the Droplet (or pushed there via Git).
  4. uv installed on the Droplet (curl -LsSf https://astral.sh/uv/install.sh | sh).
  5. Node.js 20+ installed on the Droplet (curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - && sudo apt-get install -y nodejs).

1 Add the Droplet to your Tailnet

SSH into your Droplet and run:

# Install Tailscale
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

Tailscale prints an authentication URL — open it in a browser and approve the device. Then grab the Droplet's Tailscale IP:

tailscale ip -4   # e.g. 100.64.x.x — note this down
💡 Set a stable machine name in the Tailscale admin console (e.g. concept-droplet) so you can reach it via http://concept-droplet instead of a raw IP.

2 Choose how to serve the app

Option A

Dev servers

Run Vite and uvicorn side-by-side. Fast to set up; no build step needed. Fine for personal use.

Option B — Recommended

Build + serve from FastAPI

Build the frontend once, serve everything from a single port. More stable; no Vite process to babysit.

Option A — Dev servers

Open two terminal sessions (or use tmux panes) on the Droplet.

Terminal 1 — Backend:

cd /path/to/Concept-Master
uv run uvicorn backend.main:app --reload --port 8000 --host 0.0.0.0

Terminal 2 — Frontend:

cd /path/to/Concept-Master/frontend
npm install
npx vite --host 0.0.0.0

Access the app from any Tailnet device at:

http://<tailscale-ip>:5173

Option B — Build + serve from FastAPI

Step 1. Build the frontend:

cd /path/to/Concept-Master/frontend
npm install
npm run build   # produces frontend/dist/

Step 2. Mount the static files in FastAPI. Open backend/main.py and add these lines after all /api routers are registered:

# backend/main.py
from fastapi.staticfiles import StaticFiles
import os

DIST = os.path.join(os.path.dirname(__file__), "../frontend/dist")
app.mount("/", StaticFiles(directory=DIST, html=True), name="static")
⚠️ The static mount must come last — after all /api routes — or it will catch API requests before they reach your routers.

Step 3. Start only uvicorn:

cd /path/to/Concept-Master
uv run uvicorn backend.main:app --port 8000 --host 0.0.0.0

Access the app at:

http://<tailscale-ip>:8000

3 Keep it running after you disconnect

Use tmux so the process survives SSH disconnects:

# Create a new session
tmux new -s concept

# Run your start command inside the session, then detach:
# Ctrl+B, then D

# Reattach later:
tmux attach -t concept

For a more permanent setup, create a systemd service:

# /etc/systemd/system/concept-master.service
[Unit]
Description=Concept Master
After=network.target

[Service]
User=YOUR_USER
WorkingDirectory=/path/to/Concept-Master
ExecStart=/home/YOUR_USER/.local/bin/uv run uvicorn backend.main:app --port 8000 --host 0.0.0.0
Restart=on-failure

[Install]
WantedBy=multi-user.target
# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable --now concept-master
sudo systemctl status concept-master
💡 Find the correct path to uv with which uv and substitute it into ExecStart.

4 Firewall note

Because you're using Tailscale, you don't need to open port 8000 on the Droplet's public firewall — Tailscale traffic is end-to-end encrypted and separate from the public network. If you have UFW enabled, you can leave it blocking 8000 externally and the app will still be reachable via the Tailscale IP.

ℹ️ Only SSH (port 22) needs to be open publicly — or you can go fully private and SSH in over Tailscale too, then disable the public SSH rule.
